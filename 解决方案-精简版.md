# Void 项目依赖问题 - 精简解决方案

## 🎯 核心问题

`npm install` 失败因为：
1. 原生模块需要编译 → 需要 Visual Studio 2022
2. 缺少预编译二进制镜像配置

## ✅ 已完成的修复

### 1. .npmrc 配置（已修复）

```ini
# 国内 npm 镜像
registry=https://registry.npmmirror.com

# VS2022 配置
msvs_version=2022

# Electron 镜像
disturl=https://electronjs.org/headers
electron_mirror=https://npmmirror.com/mirrors/electron/
electron_builder_binaries_mirror=https://npmmirror.com/mirrors/electron-builder-binaries/

# 原生模块镜像
node_sqlite3_binary_host_mirror=https://npmmirror.com/mirrors/
spdlog_binary_host_mirror=https://npmmirror.com/mirrors/
node_pty_binary_host_mirror=https://npmmirror.com/mirrors/

# 其他配置
target="34.3.2"
runtime="electron"
legacy-peer-deps=true
timeout=180000
```

**关键修复**:
- ✅ 移除 `disturl` 引号
- ✅ 添加国内 npm 镜像
- ✅ 添加 `msvs_version=2022`
- ✅ 添加 `spdlog` 镜像

### 2. .gitignore 更新

```gitignore
# 忽略原版本 exe
原版本exe/
VoidSetup-x64-*.exe

# 忽略临时文件
temp_*
```

## 🚀 安装步骤

### 方案 A: 全自动安装（需 VS2022 + Spectre 库）

```bash
npm install
```

### 方案 B: 分步安装（推荐）

```bash
# 步骤 1: Electron
npm install electron@34.3.2 --save-dev

# 步骤 2: 核心原生模块
npm install @vscode/sqlite3@5.1.8-vscode --save
npm install @vscode/spdlog@^0.15.0 --save
npm install node-pty@^1.1.0-beta33 --save

# 步骤 3: 其他原生模块
npm install kerberos@2.1.1 native-keymap@^3.3.5 native-watchdog@^1.4.1 --save

# 步骤 4: 剩余依赖
npm install --legacy-peer-deps
```

### 方案 C: 跳过编译（无 VS2022 时）

```bash
npm install --ignore-scripts
```

## ⚠️ VS2022 要求

**编译需要安装的组件**:
1. MSVC v143 生成工具
2. **Spectre 缓解库** (关键!)
3. Windows 11 SDK (10.0.22621.0)

**安装命令**:
```bash
# 使用 VS Installer 命令行
vs_buildtools.exe --add Microsoft.VisualStudio.Workload.VCTools --add Microsoft.VisualStudio.Component.VC.Tools.x86.x64 --add Microsoft.VisualStudio.Component.VC.ATL.Spectre --add Microsoft.VisualStudio.Component.Windows11SDK.22621
```

## 🔍 故障排查

| 问题 | 解决方案 |
|------|----------|
| `MSB8040` Spectre 错误 | 安装 VS2022 Spectre 缓解库组件 |
| `node-gyp` 找不到 VS | 设置 `msvs_version=2022` |
| 下载超时 | 使用国内镜像或增加 `--timeout=300000` |
| 模块安装失败 | `npm install <模块> --ignore-scripts` |

## 📊 原生模块优先级

| 优先级 | 模块 | 难度 |
|--------|------|------|
| 1 | electron | ⭐ 低 |
| 2 | sqlite3 | ⭐⭐ 中 |
| 3 | spdlog | ⭐⭐ 中 |
| 4 | node-pty | ⭐⭐⭐ 高 |
| 5 | kerberos/native-* | ⭐⭐⭐⭐ 很高 |

## 💡 关键结论

1. **VS2022 是必需的** - 用于编译原生模块（即使使用预编译二进制也可能需要回退编译）
2. **Spectre 缓解库是关键** - 缺少此组件会导致编译失败
3. **国内镜像加速明显** - npmmirror.com 替代官方源
4. **分步安装更可控** - 先安装核心模块，再安装其他依赖

## 🔗 快速命令

```bash
# 验证配置
npm config get registry
npm config get electron_mirror

# 完整安装（需要 VS2022）
npm install --legacy-peer-deps

# 仅安装 JS 依赖（跳过原生模块编译）
npm install --ignore-scripts

# 强制重新编译
npm rebuild
```
