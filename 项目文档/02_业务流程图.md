│                        应用启动流程                              │

    ↓
[5] 初始化工作台
    - 加载主题
    - 初始化编辑器

    src/vs/workbench/contrib/welcomeGettingStarted/browser/gettingStarted.ts

## 3. 语言选择流程

```
┌─────────────────────────────────────────────────────────────────┐
│                      语言选择业务流程                            │
└─────────────────────────────────────────────────────────────────┘

[1] 用户触发
    - 点击 🌐 Language 按钮
    - 或按 Ctrl+Shift+P → Configure Display Language
    ↓
[2] 调用语言选择器
    src/vs/workbench/contrib/welcomeGettingStarted/browser/gettingStarted.ts
    showLanguageSelector()
    ↓
[3] 获取语言列表
    ├─ ILanguagePackService.getInstalledLanguages()  # 已安装
    └─ ILanguagePackService.getAvailableLanguages() # 可用
    ↓
[4] 显示 QuickPick
    ├─ 分隔符: "Installed"
    ├─ 已安装语言列表
    ├─ 分隔符: "Available"
    └─ 可用语言列表
    ↓
[5] 用户选择语言
    ↓
[6] 设置语言
    ILocaleService.setLocale(selectedLanguage)
    ↓
[7] 提示重启
    "需要重启应用以应用语言更改"
    ↓
[8] 应用重启后生效
```

## 4. Void AI 聊天流程

```
┌─────────────────────────────────────────────────────────────────┐
│                      Void AI 聊天业务流程                        │
└─────────────────────────────────────────────────────────────────┘

[1] 用户发送消息
    - 在侧边栏输入消息
    - 点击发送或按 Enter
    ↓
[2] 前端处理
    src/vs/workbench/contrib/void/browser/voidChatService.ts
    - 创建消息对象
    - 显示在用户界面
    - 添加到消息历史
    ↓
[3] 发送到 LLM
    ├─ 如果是本地模型 → 通过 electron-main 通道
    ├─ 如果是远程模型 → 直接发送 API 请求
    └─ 加载模型配置
    ↓
[4] LLM 处理
    - 流式返回响应
    ↓
[5] 接收响应
    - 流式显示在聊天窗口
    - 支持 Markdown 渲染
    - 代码块高亮
    ↓
[6] 应用代码建议 (可选)
    ├─ 点击 Apply → 创建 DiffZone
    ├─ 显示红/绿对比
    ├─ 用户确认
    └─ 应用到文件
    ↓
[7] 消息完成
```

## 5. 代码应用流程

```
┌─────────────────────────────────────────────────────────────────┐
│                      代码应用业务流程                          │
└─────────────────────────────────────────────────────────────────┘

[1] 触发应用
    ├─ 用户点击 Apply
    ├─ LLM 调用 Edit 工具
    └─ 用户按 Cmd+K
    ↓
[2] 判断应用模式
    ├─ Fast Apply (默认)
    │   └─ 使用 Search/Replace 格式
    └─ Slow Apply
        └─ 重写整个文件
    ↓
[3] 创建 DiffZone
    src/vs/workbench/contrib/void/browser/voidEditCodeService.ts
    - 标记需要修改的行范围
    - 计算原始/修改对比
    ↓
[4] 显示差异
    - 红色: 删除的行
    - 绿色: 新增的行
    - 用户可查看对比
    ↓
[5] 用户操作
    ├─ 接受 → 应用更改到文件
    ├─ 拒绝 → 取消更改
    └─ 编辑 → 手动修改建议
    ↓
[6] 写入文件
    src/vs/workbench/contrib/void/browser/voidModelService.ts
    - 更新文本模型
    - 自动保存到磁盘
    ↓
[7] 完成
```

## 8. 多语言切换完整流程

```
┌─────────────────────────────────────────────────────────────────┐
│                    多语言切换完整业务流程                        │
└─────────────────────────────────────────────────────────────────┘

[1] 入口选择
    ├─ 欢迎页面 🌐 Language 按钮
    ├─ Ctrl+Shift+P → Configure Display Language
    └─ 设置 → 语言配置
    ↓
[2] 显示语言选择器
    src/vs/workbench/contrib/localization/common/localizationsActions.ts
    或 gettingStarted.ts showLanguageSelector()
    ↓
[3] 获取可用语言
    ILanguagePackService
    ├─ getInstalledLanguages()
    └─ getAvailableLanguages()
    ↓
[4] 显示语言列表
    IQuickInputService.createQuickPick()
    ├─ 已安装语言（分隔符 + 列表）
    └─ 可用语言（分隔符 + 列表）
    ↓
[5] 用户选择语言
    ├─ 选择已安装 → 直接应用
    └─ 选择可用 → 下载并安装语言包
    ↓
[6] 设置区域设置
    ILocaleService.setLocale(language)
    ├─ Electron: 写入 argv.json
    └─ Web: 写入 localStorage
    ↓
[7] 提示重启
    "语言设置已更改。需要重启应用以生效。"
    ↓
[8] 用户重启应用
    ↓
[9] 重启后加载
    - 读取 argv.json / localStorage
    - 加载对应语言包
    - 重新渲染界面
    ↓
[10] 多语言界面显示完成
```
