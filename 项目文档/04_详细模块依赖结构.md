## 七、架构设计原则

### 7.1 VSCode/Void 架构核心

```
1. 依赖注入 (DI) 模式
   - 所有服务通过 ServiceCollection 注册
   - 构造函数使用 @IServiceName 注入
   - 单例模式管理共享服务

2. 分层架构
   - platform/ - 平台抽象层
   - base/ - 基础工具层
   - workbench/ - 工作台层
   - editor/ - 编辑器层

3. 贡献点模式 (Contribution)
   - 功能模块通过 registerWorkbenchContribution 注册
   - 生命周期阶段控制初始化时机

4. 进程分离
   - electron-main/ - 主进程（系统访问）
   - browser/ - 渲染进程（UI）
   - common/ - 共享代码
```
