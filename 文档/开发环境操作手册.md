## 1. 环境前置条件

### 1.1 必须满足

| 条件 | 说明 | 验证命令 |
|------|------|---------|
| Node.js 20.18.2 | `.nvmrc` 指定，其他版本不保证兼容 | `node --version` |
| nvm for Windows | 切换 Node 版本 | `nvm use 20.18.2` |
| VS Build Tools 2022 | 原生模块编译（node-gyp） | — |
| Python 3.x | node-gyp 依赖 | `python --version` |

### 1.2 已知 ESM 兼容问题（必须修补）

**问题**：`node_modules/@vscode/gulp-electron/src/download.js` 用 `require()` 引入了 ESM-only 模块，Node 20 下报 `ERR_REQUIRE_ESM`。

**影响命令**：`npm run compile`、`npm run watch`、`npm run watch-client` 均失败。

**修补位置**：`node_modules/@vscode/gulp-electron/src/download.js`

**修补内容**：将以下三处 `require()` 改为懒加载 `import()`：

```javascript
// 原始（错误）
const { downloadArtifact } = require("@electron/get");
const { Octokit } = require("@octokit/rest");
const { got } = require("got");

// 修补后（懒加载）
let _electronGet, _octokit, _got;
async function loadElectronGet() {
  if (!_electronGet) _electronGet = await import("@electron/get");
  return _electronGet;
}
async function loadOctokit() {
  if (!_octokit) _octokit = await import("@octokit/rest");
  return _octokit;
}
async function loadGot() {
  if (!_got) _got = await import("got");
  return _got;
}
```

> ⚠️ 此修补在 `node_modules` 中，`npm install` 后需重新修补。项目根目录有 `patch-gulp-electron.cjs` 可一键执行修补。

## 2. 开发模式启动流程

### 2.1 关键原则

**错误方式（修改不生效）**：
```powershell
# Start-Process 不继承当前 shell 的 $env: 变量
# VSCODE_DEV=1 不会传入 Void，Void 以生产模式运行
# 生产模式不从 out/ 加载 JS，所有源码修改无效
$env:VSCODE_DEV='1'; Start-Process -FilePath '.\.build\electron\Void.exe'
```

**正确方式（修改生效）**：
```powershell
# & 操作符在当前 PowerShell 进程上下文执行，$env: 变量正确继承
$env:VSCODE_DEV='1'; & '.\.build\electron\Void.exe' .
```

### 2.2 完整启动步骤

#### 步骤 1：切换 Node 版本

```powershell
nvm use 20.18.2
```

#### 步骤 2（仅首次或 npm install 后）：修补 ESM 兼容

```powershell
node patch-gulp-electron.cjs
```

#### 步骤 3：启动增量编译（终端 1，保持运行）

```powershell
# 工作目录：项目根目录
npm run watch-client
```

**等待标志**：终端出现以下内容后才能启动 Void：
```
[HH:MM:SS] Finished compilation with 0 errors after XXXXX ms
```

> ⚠️ watch-client 首次启动会 `clean-out`（清空 out/ 目录），约 25 秒。
> 必须等编译完成后再启动 Void，否则 Void 因找不到 JS 文件而 exit code 1。

> ⚠️ `npm run watch`（全量）在中文路径下 watch-extensions 报 `spawn UNKNOWN`，
> 改用 `npm run watch-client` 仅监听核心 workbench 变化。

#### 步骤 4：启动 Void（终端 2）

```powershell
# 工作目录：项目根目录
$env:NODE_ENV='development'
$env:VSCODE_DEV='1'
$env:VSCODE_CLI='1'
$env:ELECTRON_ENABLE_LOGGING='1'
& '.\.build\electron\Void.exe' . `
  --remote-debugging-port=9222 `
  --user-data-dir ./.tmp/user-data-dev `
  --extensions-dir ./.tmp/extensions `
  2>&1 | Tee-Object -FilePath ./.tmp/void-runtime.log
```

**参数说明**：

| 参数 | 作用 |
|------|------|
| `VSCODE_DEV=1` | 开启开发模式，从 `out/` 加载 JS |
| `--remote-debugging-port=9222` | 开放 CDP 端口，供 AI 工具连接查看 GUI |
| `--user-data-dir ./.tmp/...` | 隔离用户数据，避免污染正式配置 |
| `Tee-Object` | 日志同时输出到终端和文件 |

## 3. 验证开发模式是否生效

### 3.1 通过日志文件确认

```powershell
Get-Content .\.tmp\void-runtime.log | Select-String "isDevMode"
# 期望输出：isDevMode: true
```

### 3.2 通过 CDP JSON 确认

```
GET http://localhost:9222/json
```

期望返回包含：
```json
{
  "title": "Void Dev [Administrator]",
  "url": "...workbench-dev.html"
}
```

`workbench-dev.html`（非 `workbench.html`）= dev 模式正确加载 `out/` 目录。

### 3.3 通过 Void 内 DevTools 确认

在 Void 窗口按 `Ctrl+Shift+I` 打开 DevTools → Console：
```javascript
process.env.VSCODE_DEV   // 返回 "1" 则正确
```

## 4. 修改代码并查看效果

### 4.1 闭环流程

```
1. 修改 src/**/*.ts 源码
        ↓
2. watch-client 自动检测并增量编译（5-15 秒）
   终端显示：Finished compilation with 0 errors
        ↓
3. 在 Void 窗口：Ctrl+Shift+P → "Developer: Reload Window"
        ↓
4. 界面立即反映修改，无需重启 Void
```

### 4.2 完整修改周期时间

| 阶段 | 耗时 |
|------|------|
| 保存 .ts 文件到编译完成 | 5–15 秒（增量） |
| Reload Window | < 3 秒 |
| **总计** | **约 10–20 秒** |

对比全量 `npm run compile`：约 6–8 分钟。

## 7. 常见问题排查

| 现象 | 原因 | 解决 |
|------|------|------|
| 修改代码后界面无变化 | `Start-Process` 未传递 `VSCODE_DEV=1` | 改用 `&` 操作符启动 |
| Void exit code 1 启动失败 | watch-client 还在 clean-out/编译阶段 | 等 "Finished compilation 0 errors" 后再启动 |
| `npm run compile` ERR_REQUIRE_ESM | ESM 兼容问题 | 执行 `node patch-gulp-electron.cjs` |
| `npm run watch` spawn UNKNOWN | 路径含中文字符 | 改用 `npm run watch-client` |
| CDP `/json` 返回 `[]` | Void 窗口还未完成加载 | 等待 5-10 秒后重试 |
| `isDevMode: false` 或 undefined | env 变量未传入 | 确认使用 `&` 而非 `Start-Process` |
