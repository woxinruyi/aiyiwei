### 关键规则（必须遵守）

| # | 规则 | 原因 |
|---|------|------|
| 1 | React 组件源码只改 `react/src/`，**绝不改 `src2/`** | `src2/` 是 scope-tailwind 自动生成的中间产物，会被覆盖 |
| 2 | 改完 React 组件必须执行 `node build.js` | watch-client 不会执行 scope-tailwind 和 tsup |
| 3 | `node build.js` 后必须**手动复制** bundle 到项目根 `out/` | watch-client 不会自动同步 React bundle |
| 4 | 任何 UI 修改都需要**重启 Void 窗口** | React 部分无 HMR |
| 5 | `localize('key', 'fallback')` 的 fallback 会被 NLS 覆盖 | 测试时直接用字符串替换整个 localize() |
| 6 | esbuild 会把中文编码为 `\uXXXX` | 搜索 bundle 时用英文关键词定位，不要搜中文原文 |

## 一、项目架构概览

Void 是基于 VSCode 的 AI 代码编辑器，其 UI 层由**两套独立的渲染体系**组成：

```
┌──────────────────────────────────────────────────┐
│  Electron 主进程                                  │
├──────────────────────────────────────────────────┤
│  Renderer 进程                                    │
│  ┌──────────────────────────────────────────┐    │
│  │  VSCode Workbench（TypeScript + 原生 DOM）│    │
│  │                                          │    │
│  │   ┌─────────────────────────────────┐    │    │
│  │   │  React 岛屿（Island Architecture）│    │    │
│  │   │  ┌────────┐ ┌──────┐ ┌──────┐  │    │    │
│  │   │  │Onboard.│ │ Chat │ │Sett. │  │    │    │
│  │   │  └────────┘ └──────┘ └──────┘  │    │    │
│  │   └─────────────────────────────────┘    │    │
│  └──────────────────────────────────────────┘    │
└──────────────────────────────────────────────────┘
```

### 1.1 两套 UI 体系对照

| 体系 | 技术栈 | 代表页面 | 构建方式 |
|------|--------|---------|---------|
| **VSCode 原生** | TypeScript + DOM API + `localize()` | 欢迎页(Getting Started)、编辑器、设置 | `watch-client` 自动编译 TS → JS |
| **Void React** | React + TSX + TailwindCSS + tsup | 首屏引导(Onboarding)、侧栏Chat、AI设置 | `scope-tailwind` → `tsup` → 手动复制 |

**核心区别**：VSCode 原生 UI 改 `.ts` 后 watch-client 自动编译即可；Void React UI 改 `.tsx` 后**必须执行独立的 React 打包流程**。

### 1.2 Void React 组件的"岛屿架构"

Void 新增的 AI 功能并未重写整个 Renderer，而是采用**岛屿架构**（Island Architecture）：

- 主体 UI 仍由 VSCode 原生 TypeScript 代码控制
- 每个 AI 功能（Chat、设置、引导页等）是一个**独立的 React 应用**
- 每个 React 应用打包为单独的 ESM bundle（约 1~1.5MB）
- 通过 `mountFnGenerator` 桥接层挂载到 VSCode 创建的 DOM 容器中
- VSCode 服务（主题、命令、文件等）通过依赖注入传入 React Context

## 二、React 组件构建流水线

### 2.1 三步构建流程

```
src/*.tsx ──① scope-tailwind──▶ src2/*.tsx ──② tsup──▶ out/*/index.js ──③ 复制──▶ 项目根 out/
 (开发者编辑)    (CSS class 加前缀)    (ESM bundle)         (Void 运行时加载)
```

| 步骤 | 工具 | 作用 | 输入 → 输出 |
|------|------|------|------------|
| ① | `scope-tailwind` | TailwindCSS class 加 `void-` 前缀，避免与 VSCode 样式冲突 | `src/` → `src2/` |
| ② | `tsup`（esbuild） | 编译 TSX + 打包为单文件 ESM bundle + 内联 CSS | `src2/` → `react/out/*/index.js` |
| ③ | 手动复制或 watch-client | 将 bundle 复制到 Void 运行时目录 | `react/out/` → 项目根 `out/.../react/out/` |

### 2.2 目录结构

```
src/vs/workbench/contrib/void/browser/react/
├── build.js              ← 构建入口（node build.js 或 node build.js --watch）
├── tsup.config.js        ← tsup 打包配置（入口为 src2/、输出到 out/）
├── src/                  ← 源码目录（开发者在此编辑）
│   ├── void-onboarding/
│   │   ├── index.tsx           ← 导出 mountVoidOnboarding
│   │   └── VoidOnboarding.tsx  ← 首屏 "Welcome to Void"
│   ├── sidebar-tsx/            ← Chat 侧边栏
│   ├── void-settings-tsx/      ← AI 设置面板
│   └── util/
│       ├── mountFnGenerator.tsx ← React→VSCode 桥接（ReactDOM.createRoot）
│       └── services.tsx         ← VSCode 服务注入 React Context
├── src2/                 ← scope-tailwind 中间产物（自动生成，勿手动编辑）
└── out/                  ← tsup 打包最终产物
    └── void-onboarding/
        └── index.js      ← Void 运行时实际加载的 bundle
```

### 2.3 挂载流程

```
voidOnboardingService.ts
  → registerWorkbenchContribution2(WorkbenchPhase.AfterRestored)
    → OnboardingContribution.initialize()
      → document.querySelector('.monaco-workbench')
      → 创建 div.void-onboarding-container 并 appendChild
      → import { mountVoidOnboarding } from './react/out/void-onboarding/index.js'
        → mountFnGenerator(VoidOnboarding)
          → _registerServices(accessor)  // 注入 VSCode 服务
          → ReactDOM.createRoot(container)
          → root.render(<VoidOnboarding />)
```

## 三、二次开发修改流程

### 3.1 修改 Void React UI（首屏、Chat、设置等）


## 四、与传统 React + Electron 架构的对比

| 维度 | 传统 React+Electron | Void 混合架构 |
|------|---------------------|--------------|
| **UI 所有权** | React 拥有整个 Renderer | VSCode 拥有 Renderer，React 是 "岛屿" |
| **构建流程** | 1 步（webpack/vite） | 3 步（scope-tailwind → tsup → 复制） |
| **热更新** | HMR 即时生效 | 无 HMR，需重新打包 + 重启窗口 |
| **CSS 隔离** | 无需特殊处理 | 必须 scope-tailwind 加前缀防冲突 |
| **服务访问** | Electron IPC 直连 | 通过桥接层从 VSCode DI 容器注入 |
| **修改难度** | 低（改了就生效） | 高（需理解双重构建体系） |
| **编辑器能力** | 需自行实现 | 零成本继承 VSCode 全部能力 |

### 优点

1. **零成本继承 VSCode 生态** — 编辑器、调试、Git、扩展全部可用
2. **渐进式增强** — AI 功能用 React，不破坏 VSCode 稳定性
3. **CSS 隔离** — scope-tailwind 确保 Tailwind 不污染 VSCode 原有样式
4. **按需加载** — 每个 React 岛屿独立 bundle，未使用不加载
5. **VSCode 服务直通** — React 组件可调用所有 VSCode 内部服务

### 缺点

1. **构建复杂** — 三步流水线，任何一步遗漏都会导致修改不生效
2. **无 HMR** — 每次改动需重新打包 + 重启窗口
3. **调试困难** — 1MB+ 单文件 bundle，sourcemap 支持有限
4. **双重心智模型** — 需同时理解 VSCode DI/生命周期和 React 组件模型
5. **中文编码** — esbuild 将中文编码为 `\uXXXX` Unicode 转义，搜索时需注意

## 六、踩坑记录与问题总结

### 6.1 改错页面（认知错误）

| 问题 | 详情 |
|------|------|
| **现象** | 修改了 `gettingStarted.ts` 中的标题文字，但首屏没有变化 |
| **原因** | 用户看到的首屏是 **VoidOnboarding.tsx**（React 组件，z-index=99999 覆盖层），而不是 VSCode 原生的 Getting Started 编辑器标签页。两者是完全不同的页面 |
| **教训** | **修改前必须先确认目标 UI 属于哪个体系**。可通过 DevTools 审查 DOM 元素确认 |

### 6.2 localize() 覆盖 fallback

| 问题 | 详情 |
|------|------|
| **现象** | `localize('key', "我的文字")` 中修改 fallback 参数不生效，仍显示英文 |
| **原因** | `localize()` 优先从 NLS 消息表 `globalThis._VSCODE_NLS_MESSAGES` 取值，fallback 只在 NLS 无此 key 时使用 |
| **解法** | 测试时直接用字符串替换整个 `localize()` 调用；正式修改需改 NLS 体系 |

### 6.3 React 构建链路断裂

| 问题 | 详情 |
|------|------|
| **现象** | 修改了 `VoidOnboarding.tsx`，`watch-client` 显示编译成功（0 errors），但 UI 没变化 |
| **原因** | `watch-client` 只编译 TypeScript 并复制文件到 `out/`，但**不会执行 scope-tailwind 和 tsup**。React 组件的 bundle 仍是旧版本 |
| **解法** | 必须在 `react/` 目录执行 `node build.js`，然后手动复制 bundle |

### 6.4 esbuild 中文编码

| 问题 | 详情 |
|------|------|
| **现象** | 用 `Select-String` 搜索中文 "欢迎来到编辑器"，在 bundle 中 Count=0 |
| **原因** | esbuild/tsup 将中文字符编码为 Unicode 转义序列 `\u6B22\u8FCE\u6765\u5230\u7F16\u8F91\u5668`，运行时正常显示中文，但文件中不包含原始中文字符 |
| **解法** | 搜索时使用 `Welcome to Void` 等英文关键词定位附近行，或搜索 Unicode 转义形式 |

### 6.5 watch-client 不复制 React bundle

| 问题 | 详情 |
|------|------|
| **现象** | `node build.js` 执行成功，`react/out/index.js` 包含修改，但项目根 `out/.../index.js` 仍是旧版 |
| **原因** | watch-client 的 gulp 任务复制 `.ts`/`.tsx` 源码但不一定会重新复制 `out/` 中的 `.js` bundle |
| **解法** | 手动执行 `Copy-Item` 将 bundle 复制到项目根 `out/` 对应目录 |

## 八、总结

1. **修改前先定位体系** — 确认目标 UI 是 VSCode 原生还是 Void React
2. **React 修改必须走完整打包** — `scope-tailwind → tsup → 手动复制 bundle`
3. **推荐双终端开发** — `watch-client` + `build.js --watch` 并行运行
4. **任何 UI 修改都需重启窗口** — React 部分无 HMR 支持
5. **验证时注意 esbuild 编码** — 中文会被转为 `\uXXXX`，搜索需用英文或转义形式
6. **测试 localize 文本时直接用字符串** — 不要依赖 fallback 参数
