# Void 项目运行方式文档

## 项目简介

Void 是一个开源的 Cursor 替代品，基于 VS Code fork 而来。它允许在代码库上使用 AI 代理，检查点和可视化更改，并支持本地托管任何模型。Void 直接向提供商发送消息，不保留用户数据。

## 技术栈

- **前端框架**: React 19.1.0
- **开发语言**: TypeScript 5.8.0
- **桌面应用**: Electron 34.3.2
- **构建工具**: Gulp, Webpack
- **样式**: Tailwind CSS
- **包管理**: npm

## 运行方式

### 方式一：开发者模式（推荐）

1. **初始化开发者模式**
   - 在 VSCode 中打开项目
   - 按下快捷键启动构建任务：
     - Windows: `Ctrl+Shift+B`
     - macOS: `Cmd+Shift+B`
     - Linux: `Ctrl+Shift+B`
   - 等待构建完成（约 5 分钟，当 3 个加载器中有 2 个变为对勾时完成）

2. **启动开发者模式窗口**
   - Windows: `./scripts/code.bat`
   - macOS/Linux: `./scripts/code.sh`

3. **开发调试**
   - 修改代码后，在开发者窗口中按 `Ctrl+R`（macOS: `Cmd+R`）重新加载
   - 或按 `Ctrl+Shift+P`，选择 "Reload Window"
   - 可选参数：`--user-data-dir ./.tmp/user-data --extensions-dir ./.tmp/extensions`
     - 使用此参数后，可通过删除 `.tmp` 文件夹重置 IDE 设置

4. **停止构建脚本**
   - 在终端中按 `Ctrl+D`（不要使用 `Ctrl+C`，因为脚本会在后台继续运行）

### 方式二：终端构建

如果不想在 VSCode 中构建，可以使用终端命令：

```bash
npm run watch
```

构建完成时会看到类似输出：
```
[watch-extensions] [00:37:39] Finished compilation extensions with 0 errors after 19303 ms
[watch-client    ] [00:38:06] Finished compilation with 0 errors after 46248 ms
[watch-client    ] [00:38:07] Starting compilation...
[watch-client    ] [00:38:07] Finished compilation with 0 errors after 5 ms
```

### 方式三：构建本地可执行文件（不推荐）

构建本地可执行文件需要约 25 分钟，通常不推荐这样做。

#### macOS
```bash
# Apple Silicon（最常见）
npm run gulp vscode-darwin-arm64

# Intel
npm run gulp vscode-darwin-x64
```

#### Windows
```bash
# x64（最常见）
npm run gulp vscode-win32-x64

# ARM64
npm run gulp vscode-win32-arm64
```

#### Linux
```bash
# x64（最常见）
npm run gulp vscode-linux-x64

# ARM64
npm run gulp vscode-linux-arm64
```

构建完成后，可执行文件将位于 `void/` 文件夹之外，例如：
```
workspace/
├── void/                    # Void 源代码
└── VSCode-darwin-arm64/     # 生成的可执行文件
```

## 常见问题解决

1. **确保 Node.js 版本正确**
   - 使用 nvm 安装指定版本：`nvm install` && `nvm use`

2. **路径不能包含空格**
   - 确保项目文件夹路径中没有空格

3. **动态导入错误**
   - 如果出现 `TypeError: Failed to fetch dynamically imported module`，确保所有导入都以 `.js` 结尾

4. **React 相关错误**
   - 尝试运行：`NODE_OPTIONS="--max-old-space-size=8192" npm run buildreact`

5. **样式缺失**
   - 等待几秒钟后重新加载

6. **libtool 错误（macOS）**
   - 确保使用 GNU libtool 而不是 BSD libtool

7. **SUID sandbox 错误（Linux/macOS）**
   ```bash
   sudo chown root:root .build/electron/chrome-sandbox
   sudo chmod 4755 .build/electron/chrome-sandbox
   ```
